# 記帳米粒 ｜ 系統說明文件

> 一個以 LINE Bot 為核心的記帳助理，搭配監控後台（給使用者看自己的帳）與中控台（給管理者維運）。

---

## 一、介紹

「記帳米粒」是一套由三個服務組成的 LINE 記帳系統：

| 服務 | 說明 | 使用對象 |
|---|---|---|
| **LINE bot 主程式**（`main.py` + `app/`） | 使用者在 LINE 對話中互動的核心邏輯，負責記帳、開團核銷、旅行規劃、繳費提醒等所有功能 | 所有使用者 |
| **監控後台**（`index.html`） | 透過 LINE LIFF 開啟的網頁，讓使用者查看自己（或群組）的記帳明細、圖表、進行資料修正 | 一般使用者 |
| **中控台 admin-panel**（`main.py` + `admin.html`） | 獨立部署的後台管理系統，讓管理者調整關鍵字、敏感詞、功能開關、發送公告、瀏覽原始資料庫等 | 系統管理者 |

三個服務共用同一個 MySQL 資料庫，各自獨立部署、獨立重啟，互不影響。

### 技術架構

- **後端框架**：FastAPI（Python）
- **資料庫**：MySQL（連線池 PooledDB）
- **對話能力**：Google Gemini（`gemini-2.5-flash`）
- **通訊**：LINE Messaging API（Webhook + LIFF）
- **前端**：純 HTML + Tailwind CDN + Chart.js（無需建置流程）

## 二、正式功能使用說明（免密碼，所有人可用）

### 2.1 個人記帳

| 指令 | 說明 |
|---|---|
| `項目 金額` | 快速記一筆支出，例如：`早餐 80` |
| `收入 項目 金額` 或 `+項目 金額` | 記一筆收入，例如：`收入 薪水 30000` |
| 自然語言（例如：「今天買咖啡花了150元」） | 交給 AI 判讀，自動萃取項目與金額 |

**自動分類**：系統會依品項自動歸類到 12 種常見分類之一（餐飲、交通、購物、娛樂消遣、居家生活、醫療保健、日用品、水電通訊、教育學習、保險、人情社交、儲蓄、其他）。判斷方式是先比對本地關鍵字庫，比對不到才呼叫 AI，兼顧速度與準確度。

### 2.2 群組功能

| 指令 | 說明 |
|---|---|
| `@記帳米粒 開團` | 開啟揪團模式，產生 4 碼團單號 |
| `品項 金額` 或 `@成員 品項 金額`（揪團模式中） | 代點單 |
| `@記帳米粒 結單` | 結束揪團，鎖定明細，產生歷史訂單 |
| `品項 金額`（一般模式中） | 系統會詢問「均分／@tag／指定金額／跳過」四選一決定怎麼分攤 |
| `@記帳米粒 申請核銷 #單號` | 解鎖核銷模式 |
| `@已還 100`（核銷模式中） | 登記某人已核銷金額，系統會自動防呆（不可溢繳、不可重複核銷） |
| `結算結束` | 關閉核銷模式，回到一般模式 |

**群組團單分攤四選一**：

1. 回覆「均分」→ 平分給已知的群組成員
2. tag 出實際分攤的人（例如 `@小明 @小華`）→ 平分給這些人
3. tag 並加金額（例如 `@小明 100 @小華 50`）→ 指定每人分攤的確切金額
4. 回覆「跳過」→ 記一般花費，不分攤

### 2.3 其他常用指令

| 指令 | 說明 |
|---|---|
| `使用說明` / `怎麼用` / `功能` | 顯示快速上手指南 |
| `查帳` / `報表` / `網址` | 群組版取得監控後台網址；個人版導引至圖文選單 |

### 2.4 監控後台（`index.html`）

透過 LINE 圖文選單或連結開啟，個人版與群組版顯示內容不同：

- **監控首頁**：本月收支總和、收入、現金支出、支付工具支出（使用率最高者）、支出與支付工具比例圖、當日即時明細（群組版顯示團購訂單金額、群組分類比例圖）
- **明細查詢**：可依日期區間、收支類型、支付方式篩選，選定支付方式後會在上方顯示該類型總和；記錄可線上編輯/刪除
- **團單核銷**（群組限定）：歷史揪團訂單列表，可微調品項、催款明細一目了然
- **旅行規劃**：旅行的查看與增減修正（詳見測試功能）
- **理財工具**（個人限定）：繳費、存錢、支付方式的新增/編輯/刪除（詳見測試功能）

### 2.5 敏感詞過濾

由中控台設定敏感詞與「情境說明」，機器人不是單純比對文字是否出現，而是把詞彙與情境說明交給 AI 判斷語意是否符合（能抓到換句話說、隱晦講法等變形），命中就回覆制式拒答訊息。

### 2.6 群組成員抓取

群組內手動輸入@記帳米粒 抓取成員 @小明 @小華 @小美 @小陳...，米粒會將群內成員登記至資料庫，因官方line限制，暫時無法自動抓取成員。

## 三、測試功能使用說明（需密碼開通）

以下功能目前僅供測試，觸發對應關鍵字後，機器人會要求輸入測試密碼，驗證成功後開通 **16 小時**（可由管理者調整時數），效期一到自動失效。**同一時間同一個使用者/群組只能開啟一種測試模式**，開新的會自動關閉舊的。

若中控台把某功能設為「維護中」，觸發時會直接收到維護訊息，無法開通；設為「Beta」則不影響使用，只是在監控後台圖示上會標示提醒。

### 3.1 旅行模式

輸入 **`旅行模式`** 開始。

```
旅行模式
  → 詢問出發時間（格式：2026-07-19 19:00 或 2026/7/19 1900，也可口語描述由AI判讀）
  → 詢問回程時間
  → 逐則輸入想去的地點名稱（不用附時間，米粒會協助搜尋地址）
  → 輸入「結束」
  → 米粒安排拜訪順序與時間，直接給結果
  → 回覆「確定」寫入資料庫，或直接描述修改內容（例如：把第2項改成15:00、刪除第3項）
  → 確定後以出發日期作為旅行單號（例如 #20260719）
```

- 隨時可輸入 `取消旅行` 中止規劃
- 完成後可輸入 `旅行修改 單號` 重新進入編輯
- 出發前 **45 分鐘**會主動推播提醒，並詢問是否要記錄花費（回覆「有」在群組會自動開團登記，個人則提示直接記帳）
- 監控後台「旅行規劃」頁可查看、新增、編輯、刪除旅行與行程地點

### 3.2 收據辨識

輸入 **`收據辨識`** 開通後，直接傳送收據照片：

- **個人版**：AI 辨識品項與金額後，直接分項記入個人帳本（自動分類）
- **群組版**：辨識完成後詢問「請幫這筆團單取個名稱」，輸入名稱後直接寫入團單資料庫（名稱、單號、品項），完成登記；分攤細節請至監控後台的「團單核銷」頁調整
- 辨識失敗會明確告知原因（下載逾時、AI 服務串接問題、辨識不到品項等）

### 3.3 繳費功能（個人限定）

輸入 **`繳費功能`** 開通後：

| 指令 | 說明 |
|---|---|
| `新增繳費 帳單名稱 金額 到期日 [期數]` | 登記繳費項目，期數選填，不填代表每月持續 |
| `查看繳費` | 列出所有登記中的繳費項目與繳費狀態 |
| `帳單名稱 金額`（跟一般快速記帳同格式） | 若符合未繳費項目，直接核銷 |

- 到期日**前 2 天**會主動推播提醒
- 核銷成功會同時記一筆一般支出（月支出總額會反映這筆繳費），並依分期期數自動遞減、推進下期到期日；期數歸零則該帳單狀態變為「已完成」
- 監控後台「理財工具」頁可增減修正繳費項目；**刪除核銷紀錄會自動把繳費狀態還原為未繳費**（含到期日、期數一併還原），同時刪除當初一起寫入的那筆支出

### 3.4 存錢功能（個人限定）

輸入 **`存錢功能`** 開通後：

| 指令 | 說明 |
|---|---|
| `新增撲滿 名稱 [目標金額]` | 建立存錢筒，最多 6 個，目標金額選填 |
| `存錢 撲滿名稱 金額` | 存款，同時記一筆分類「儲蓄」的一般支出 |
| `查看存錢` | 列出所有存錢筒與目前餘額（有目標的會顯示達成百分比） |

- 達到目標金額時會主動恭喜（只觸發一次）
- 監控後台「理財工具」頁可增減修正存錢筒，包含直接手動調整餘額

### 3.5 支付方式（個人限定）

輸入 **`支付方式`** 開通後，固定格式 **「項目 金額 支付方式」**（依空白分隔，最後一段須符合已登記的支付方式）：

```
早餐 60 LINE PAY
```

- 支付方式須先在監控後台「理財工具」頁登記，未登記過的名稱不會被辨識，一律歸類為現金
- 命中格式的記帳會標記支付方式，可在監控後台「明細查詢」依支付方式篩選、查看總和

## 附註

- 測試限定功能的密碼、開通時數由環境變數 `TEST_MODE_PASSWORD`、`TEST_MODE_HOURS` 設定
- 所有中控台可調整的設定（機器人開關、AI 人格、跑馬燈、功能狀態等）皆存於 `bot_settings` / `feature_switches` 資料表，變更後 bot 端最多 **5 秒**（跑馬燈為 **30 秒**）內自動生效，不需要重啟服務
- 資料庫異動記錄於各版本對應的 `migration_v*.sql`，部署新版本前請依序執行尚未套用過的 migration
